<p>In Xamarin Forms development, more than 90% code is shared across platforms since both the UI and business logic is shared. Dependency services and Renderers can be used to write platform specific code to access functionality not available in Forms.</p>
<p>Xamarin.Forms is best for:</p>
<ul>
<li>Apps that require little platform-specific functionality</li>
<li>Apps where code sharing is more important than custom UI</li>
<li>Developers comfortable with XAML</li>
</ul>

<p>Also called Xamarin Native development, is a xamarin project in which the business logic is shared while the UI's are seperate for each platform. This allows for greater control over the App's user experience.</p>
<p>Xamarin.iOS &amp; Xamarin.Android are best for:</p>
<ul>
<li>Apps with interactions that require native behavior</li>
<li>Apps that use many platform-specific APIs</li>
<li>Apps where custom UI is more important than code sharing</li>
</ul>
